## FAZ 3 – AI İLE GELİŞTİRME SÜRECİ

# Bu Fazda Yaptıklarım
Bu fazda alışveriş sepeti sistemi üzerinde behavioral design pattern’leri uyguladım. 
Strategy Pattern ve Observer Pattern kullanarak sistemin daha esnek ve genişletilebilir hale getirdim.
Strategy Pattern sayesinde farklı indirim algoritmalarını çalışma zamanında değiştirebilir bir yapı oluşturdum. Böylece sisteme yeni indirim türleri eklemek kolaylaştı.
Observer Pattern ile sepet üzerinde yapılan değişiklikleri takip ettim. Ürün eklendiğinde veya sepet güncellendiğinde sistemin bunu algılayıp bildirim ve loglama yapmasını sağladım.
Bu süreçte GitHub üzerinde branch kullanımı, pull request oluşturma ve merge işlemleri yaptım. Ayrıca GitHub Actions kullanarak basit bir CI pipeline ekledim.

# Bu Süreçte Öğrendiklerim
Bu fazda design pattern’lerin gerçek projelerde nasıl kullanıldığını daha iyi öğrendim. Open/Closed Principle’ın kodun sürdürülebilirliği açısından neden önemli olduğunu gördüm.
Ayrıca Git ve GitHub süreçlerinde pratik kazandım. Branch yönetimi, PR oluşturma, merge işlemleri ve CI pipeline mantığını uygulamalı şekilde deneyimledim.

# AI Olmadan Bu Faz Ne Kadar Sürerdi?
AI desteği olmadan bu fazın tamamlanması muhtemelen birkaç gün sürebilirdi. Çünkü design pattern’lerin doğru şekilde uygulanması, sınıflar arası ilişkilerin kurulması ve GitHub 
süreçlerinin hatasız yönetilmesi daha fazla zaman alırdı.

# AI Bu Süreçte Nerede Yanılttı?
AI bazı durumlarda dosya yolu ve proje yapısı konusunda yanlış yönlendirmeler yaptı. Özellikle diagram path kısmında hatalar oluştu ancak bunlar düzelterek doğru yapıya ulaştım.
